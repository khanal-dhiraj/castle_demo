# GitHub Action as v1 trigger + runtime

The scaffold ships `.github/workflows/insta-engineer.yml` triggered on `issues.labeled` (label = `insta-engineer`), running on a GH-hosted `ubuntu-latest` runner with Docker-in-Docker for the sandcastle sandbox. Picked over an always-on webhook worker (Fly/Modal/Render) or Vercel Firecracker because the trigger ships in the repo — a stranger forks the scaffold, adds two secrets, labels an issue, and it works. No new infra, no GH App registration, no webhook endpoint to host. The trade-off is the 6-hour job ceiling and 3–5 min cold-start for the Docker image build; both are tolerable for v1 and signal "ticket too big" rather than infra failure if hit. Graduation path is mechanical — replace the workflow with a webhook handler that runs the same `npx tsx .sandcastle/main.mts`; the orchestrator code doesn't change.
